# Global Rules

## Code Quality

- Never use type `any` in new code. Use proper types, generics, or `unknown` with type guards.
- Do not write verbose comments explaining what code does. Write readable code with clear, descriptive function and variable names. Comments should only explain _why_, not _what_.

## Git

- Never make changes directly on the `main` branch. Always create or switch to a feature branch first.
- Never add yourself as a co-author in commit messages.
